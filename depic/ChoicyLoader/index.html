ChoicyLoader

Description

  • A hacky workaround to always load Choicy first. Renames Substrate / Substitute related files, use at your own risk.
    On some devices, Substrate / Substitute does not load tweaks alphabetically, so Choicy cannot load first and therefore not prevent other tweaks from loading.
    This package renames /usr/lib/substrate/SubstrateLoader.dylib or /usr/lib/substitute-loader.dylib to SubstrateLoader_orig.dylib / substitute_loader_orig.dylib, it then places a symlink to /usr/lib/ChoicyLoader.dylib in place of the original file.
    ChoicyLoader loads Choicy first and then the SubstrateLoader_orig.dylib / substitute_loader_orig.dylib file, resulting in Choicy getting loaded first no matter what.
    It also adds a dpkg trigger to automatically reapply everything when Substrate or Substitute updates and also reverts everything when uninstalling.
